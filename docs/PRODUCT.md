# KOVE OPS product guide

## Purpose

KOVE OPS gives an operations lead a calm, high-signal view of what requires attention now. It combines maintenance demand, asset reliability, response performance, inventory risk, and team activity without forcing the user to reconcile multiple systems.

## Primary workflow

1. Review the four operational metrics for exceptions.
2. Inspect priority work by urgency, owner, due time, and status.
3. Use global search to narrow the queue by work-order ID, asset, site, or team member.
4. Compare completion, preventive maintenance, and inventory health.
5. Follow live activity to understand recent changes.
6. Start a work order from the primary action.

## Interface behavior

- Workspace navigation updates the page context and collapses automatically on mobile.
- Search filters the priority-work dataset immediately and provides an explicit empty state.
- Rows and utility actions provide lightweight confirmation feedback.
- The health time-range control is a native select for reliable keyboard and touch behavior.
- At mobile widths, dense secondary columns are hidden while task identity and status stay visible.

## Visual system

The design uses deep industrial green for navigation and action, warm off-white surfaces to reduce glare, coral and amber for exceptions, and restrained blue for informational state. Serif display type separates decision-level information from compact operational labels.
